Cascading Form

Dependent dropdowns with server-side repopulation

Try it: Select a category to see the products list update.
When category changes, Datastar POSTs the form and patches the response via SSE. The server re-renders the form fragment with updated product choices.

How it works


    
1. Declarative Form Definition (no __init__ needed!)

    
class CascadingForm(ReactiveForm):
    category = ReactiveChoiceField(
        choices_from=get_categories,
        value_field="id",
        label_field="name",
        empty_choice="-- Select Category --",
    )
    product = ReactiveChoiceField(
        choices_from=get_products_for_category,
        depends_on=["category"],
        value_field="id",
        label_template="{name} (${price})",
        empty_choice="-- Select Product --",
        empty_choice_no_parent="-- Select Category First --",
    )


    
2. Template with Datastar POST

    
data-on:change="@post('...', {contentType: 'form'})"

    
contentType: 'form' sends form data with CSRF token.


    
3. View returns SSE fragment

    
if request.headers.get("Datastar-Request") == "true":
    form = CascadingForm(initial={...})  # unbound for repopulate
    form_html = render_to_string("_fragment.html", {"form": form})
    return DatastarResponse(patch_elements(form_html))
