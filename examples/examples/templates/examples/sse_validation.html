How it works

Instead of a full page reload on validation errors, the form submits via Datastar @post and the server responds with an SSE event that patches just the form HTML.

1. Template

{% render_reactive_form form
    "Register"
    action=action_url %}

The action parameter switches the form from native submit to Datastar @post.

2. View

form = MyForm(request.POST)
response = reactive_form_response(
    request, form,
    "_fragment.html",
    success_url="/done/",
)
if response:
    return response

One function handles SSE patch (errors) and SSE redirect (success).

3. What the server returns

On validation error:

event: patch-elements
data: <div id="reactive-form-container">
data:   <form ...>...errors...</form>
data: </div>

On success:

event: redirect
data: /success/

Backend validations used

Username Uniqueness check (simulated DB query with 100ms latency)
Email Cross-field: business accounts can't use free email providers
VAT Number EU VAT format validation (would call VIES API in production)
Coupon Code Server-side lookup against coupon database
